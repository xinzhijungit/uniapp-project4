## 1. 产品概述

### 1.1 功能定位

"智能问答"功能是"数据研判"模块的核心组成部分，旨在通过AI大语言模型与数据查询引擎的结合，为用户提供自然语言交互的数据查询与分析能力。用户可通过自然语言提问，系统自动解析问题并返回图表、统计数据和智能分析建议，实现数据查询的智能化、便捷化。

### 1.2 核心价值

*   **自然交互**：支持自然语言提问，降低数据查询门槛，无需掌握SQL或复杂查询语法。
*   **智能分析**：自动生成图表可视化和文字分析，提供关键发现和延伸解读。
*   **多轮对话**：支持上下文关联的多轮对话，实现渐进式深入分析。
*   **数据驱动**：基于业务数据库（如接警单表）进行实时数据分析，确保数据准确性和时效性。

### 1.3 功能模块组成

1.  **智能问答入口**：对话界面，包含AI助手展示、输入框、数据集选择、多轮模式切换。
2.  **对话交互**：支持自然语言输入、语音输入、问题提交和结果展示。
3.  **智能分析输出**：图表展示、关键发现、延伸解读等多种形式的分析结果。

## 4. 功能详细说明

### 4.1 模块一：智能问答入口页

#### 4.1.1 智能问答主页面

*   **功能路径**：数据研判 > 智能问答

*   **页面组成**：AI助手区域、对话历史区、输入区、功能按钮区。

*   **功能点与交互说明**：

| 模块         | 元素                    | 类型/规则                                  | 交互说明                                                                      |
| :----------- | :-------------------- | :------------------------------------- | :------------------------------------------------------------------------ |
| **AI助手区域** | 头像、欢迎语                | 静态展示                                   | 显示"智慧公安问数管家1.2"头像和欢迎语"Hi~ 我是 智慧公安问数管家1.2"。 |
| **对话历史区** | 对话消息列表                | 消息气泡                                   | 展示用户提问和AI回答的历史记录，支持滚动查看。                                       |
| **输入区**    | 输入框                  | 文本输入框                                  | 提示"请输入您的问题，可通过shift+Enter换行"，支持多行输入。                           |
| **功能按钮区** | 多轮按钮                 | 开关按钮                                   | 点击切换多轮模式，开启后支持上下文关联对话。                                         |
|              | 数据集范围选择              | 下拉选择框                                  | 选项：全部数据集、特定数据集（如接警单数据、预警数据等）。默认"数据集范围(全部)"。            |
|              | 数据总结选择               | 下拉选择框                                  | 选项：自动总结、详细分析、仅图表等。用于控制AI回答的输出形式。                            |
|              | 语音输入按钮               | 图标按钮                                   | 点击启用语音输入功能。                                                              |
|              | 发送按钮                 | 图标按钮                                   | 点击提交问题，支持Enter快捷键提交（shift+Enter换行）。                               |

#### 4.1.2 对话交互流程

*   **问题提交**：
    1. 用户在输入框输入问题。
    2. 选择数据集范围（可选，默认为全部）。
    3. 点击发送按钮或按Enter键提交。
    4. 系统显示"思考中..."状态。
    5. AI解析问题，查询数据库，生成分析结果。
    6. 展示回答结果（图表+文字分析）。

*   **多轮对话模式**：
    1. 点击"多轮"按钮开启多轮模式。
    2. 后续提问会基于历史对话上下文进行关联分析。
    3. 系统自动识别上下文引用（如"这个趋势的原因是什么？"中的"这个趋势"）。

### 4.2 模块二：智能分析输出

#### 4.2.1 图表展示

*   **功能描述**：根据用户问题自动生成可视化图表。

*   **支持的图表类型**：
    *   **折线图**：用于展示趋势变化（如工单趋势）。
    *   **柱状图**：用于对比分析。
    *   **饼图**：用于占比分析。
    *   **表格**：用于详细数据展示。

*   **功能点与交互说明**：

| 模块         | 元素                    | 说明                                                                 |
| :----------- | :-------------------- | :----------------------------------------------------------------- |
| **图表工具栏** | 图表类型切换                | 支持切换折线图、柱状图、表格等展示方式。                                          |
|              | 分析参数设置                | 点击展开参数配置，可调整时间范围、数据维度等。                                      |
|              | 过程追溯                  | 点击可查看数据查询过程和SQL语句。                                               |
|              | 刷新按钮                  | 重新执行查询，获取最新数据。                                                   |

#### 4.2.2 关键发现与延伸解读

*   **功能描述**：AI自动分析数据，生成关键发现和延伸解读。

*   **输出内容**：
    *   **关键发现**：基于数据分析得出的核心结论。
    *   **延伸解读**：基于发现提出的进一步分析建议和行动建议。

*   **功能点与交互说明**：

| 模块         | 元素                    | 说明                                                                 |
| :----------- | :-------------------- | :----------------------------------------------------------------- |
| **关键发现**   | 列表展示                  | 以项目符号形式展示关键数据发现，如"2026年1月至5月工单数量呈现波动趋势"。                    |
| **延伸解读**   | 列表展示                  | 基于关键发现提供的深入分析和建议，如"建议分析5月工单数量骤降的原因"。                      |
| **互动反馈**   | 点赞、点踩、分享、收藏按钮        | 用户可对回答进行评价和分享。                                                   |

## 5. 非功能性需求

### 5.1 性能要求

*   问题提交后，AI响应时间应在5秒以内（复杂查询可适当延长）。
*   图表渲染时间应在1秒以内。
*   支持至少50个并发用户同时操作无明显延迟。

### 5.2 安全性

*   所有用户操作需基于现有的RBAC权限系统进行校验。
*   查询请求需进行参数校验，防止SQL注入攻击。
*   用户间的数据需进行隔离，用户只能查看其权限范围内的数据。
*   对话历史需进行加密存储，保护用户隐私。

### 5.3 易用性

*   页面布局和交互遵循平台统一设计规范。
*   提供常见问题示例，引导用户正确提问。
*   支持语音输入，提升操作便捷性。
*   回答结果清晰可读，图表直观易懂。

## 6. 后台数据表（需新建）

### 6.1 对话记录表

```sql
CREATE TABLE `qa_conversation` (
`id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键ID',
`user_id` varchar(64) NOT NULL COMMENT '用户ID',
`user_name` varchar(256) DEFAULT NULL COMMENT '用户姓名',
`question` text NOT NULL COMMENT '用户问题',
`answer` text COMMENT 'AI回答',
`data_scope` varchar(100) DEFAULT '全部' COMMENT '数据集范围',
`conversation_mode` varchar(20) DEFAULT 'SINGLE' COMMENT '对话模式: SINGLE(单轮), MULTI(多轮)',
`context_id` bigint DEFAULT NULL COMMENT '关联的上下文对话ID',
`status` varchar(20) NOT NULL DEFAULT 'COMPLETED' COMMENT '状态: PROCESSING(处理中), COMPLETED(已完成), FAILED(失败)',
`error_message` text COMMENT '错误信息',
`create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
`update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
PRIMARY KEY (`id`),
KEY `idx_user_id` (`user_id`),
KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='智能问答对话记录表';
```

### 6.2 图表配置表

```sql
CREATE TABLE `qa_chart_config` (
`id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键ID',
`conversation_id` bigint NOT NULL COMMENT '关联的对话ID',
`chart_type` varchar(20) NOT NULL COMMENT '图表类型: LINE(折线图), BAR(柱状图), PIE(饼图), TABLE(表格)',
`chart_title` varchar(200) COMMENT '图表标题',
`chart_config` json COMMENT '图表配置参数',
`create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
PRIMARY KEY (`id`),
KEY `idx_conversation_id` (`conversation_id`),
CONSTRAINT `fk_chart_conversation` FOREIGN KEY (`conversation_id`) REFERENCES `qa_conversation` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='图表配置表';
```

### 6.3 关键发现表

```sql
CREATE TABLE `qa_key_finding` (
`id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键ID',
`conversation_id` bigint NOT NULL COMMENT '关联的对话ID',
`finding_type` varchar(20) NOT NULL COMMENT '发现类型: KEY(关键发现), EXTENSION(延伸解读)',
`content` text NOT NULL COMMENT '发现内容',
`relevance_score` decimal(5,2) DEFAULT NULL COMMENT '相关度评分',
`create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
PRIMARY KEY (`id`),
KEY `idx_conversation_id` (`conversation_id`),
CONSTRAINT `fk_finding_conversation` FOREIGN KEY (`conversation_id`) REFERENCES `qa_conversation` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='关键发现表';
```
